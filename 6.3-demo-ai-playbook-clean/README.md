# 6.3 Your AI Development Playbook — Clean Version

## Overview

This is the **starting point** for the capstone lesson. You'll design your personal AI development workflow — a decision framework for when to use which pattern, tuned to how you work.

## What You'll Build

1. A personal **decision framework** — which AI pattern for which situation
2. A reusable **template library** — specs, skills, commands, and agent configs
3. A **feedback loop** — how to evaluate and improve your workflow over time

## Getting Started

1. Read the completed version in `6.3-demo-ai-playbook/README.md` for the full playbook framework
2. Review the decision checklist to understand which patterns fit which situations
3. Start building your own template library based on the course modules you found most useful
4. Set up a pattern log to track what works for your projects

## Key Question to Answer

For each new task, ask yourself:
- Can I describe the output visually? → Vibe coding (Module 1)
- Is this a repeating pattern? → Write a skill (Module 2)
- Do I need external data? → MCP server (Module 3)
- Too big for one context window? → Break into phases (Module 4)
- Can concerns be separated? → Subagents or teams (Module 5)
- Need to validate quality? → AI-powered testing (Module 6)
