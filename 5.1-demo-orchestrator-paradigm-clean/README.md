# 5.1 From Conductor to Orchestrator — Clean Version

## Overview

This is the **starting point** for the Orchestrator Paradigm lesson. This is a conceptual module — no code to build, but essential context for understanding the shift from single-agent to multi-agent development.

## What You'll Learn

- The difference between the **conductor** paradigm (one agent, step-by-step) and the **orchestrator** paradigm (multiple agents, parallel execution)
- Why single-agent approaches hit a ceiling for complex projects
- A survey of modern orchestrator tools: GitHub Copilot Coding Agent, Google Jules, OpenAI Codex, Claude Code for Web, Cursor Background Agents

## Key Concepts

### Conductor (What You've Done So Far)
- One AI agent at a time
- Guide it step-by-step or use autonomous patterns (RALPH, Beads)
- Enhance with Skills, MCP, and Hooks
- Bounded by one context window and sequential execution

### Orchestrator (What's Next)
- Oversee a **team** of autonomous agents working in parallel
- Set high-level goals, define tasks, let agents execute independently
- Review completed work as pull requests
- Like a tech lead delegating to multiple developers
